## What is Business Source License 1.1?

Business Source License 1.1 is a source-available license that:

- Makes source code publicly accessible for viewing, learning, and development
- Allows free use for non-production purposes (evaluation, testing, development)
- Requires a commercial license for production deployments and commercial use
- Automatically converts to an open source license (AGPL v3.0) after a specified Change Date

This approach balances:
- **Source Transparency**: Full code access builds developer trust and enables learning
- **Commercial Protection**: Production use requires licensing, preventing exploitation
- **Future Flexibility**: Automatic conversion to AGPL enables eventual open source benefits

## Frequently Asked Questions

### General Questions

**Q: Is jPulse open source?**
A: jPulse uses Business Source License 1.1, which is source-available (not OSI-approved open source). The source code is fully accessible, but production use requires a commercial license until the Change Date (2030-01-01), when it converts to AGPL v3.0 (full open source).

**Q: Can I use jPulse for free?**
A: Yes, for evaluation, development, testing, and learning. Production use requires a commercial license.

**Q: What is "production use"?**
A: Production use means any deployment that serves end users, customers, or clients, or handles real business operations. This includes internal tools used by employees in production.

**Q: Can I contribute to jPulse?**
A: Yes! Contributions are welcome. By contributing, you grant the licensor the right to use your contributions under BSL 1.1 and future licenses (including AGPL v3.0).

**Q: Do I need a license for development/staging?**
A: No, development and staging environments are free under BSL 1.1.

**Q: Can I use jPulse in my SaaS product?**
A: SaaS products require a commercial license. Contact [team@jpulse.net](mailto:team@jpulse.net) for licensing options.

**Q: What if I'm building an open source application?**
