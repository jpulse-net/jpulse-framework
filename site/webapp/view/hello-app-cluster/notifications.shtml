üéì Learning: Client-Side Broadcasting Pattern

This demo shows the simplest clustering pattern - direct client-to-Redis broadcasting without server-side logic.

How it works:
  1. Client sends: JavaScript calls jPulse.appCluster.broadcast.publish()
  2. Redis distributes: Message goes to all connected server instances
  3. All clients receive: Every browser tab gets the notification instantly
  4. No persistence: Messages are ephemeral (not saved to database)
üí° Perfect for: Status updates, alerts, chat messages, real-time announcements
