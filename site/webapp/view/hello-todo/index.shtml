üìñ Model-View-Controller (MVC) Pattern

Welcome to the To-Do MVC Demo!

This demo teaches you how to build a traditional Multi-Page Application (MPA) using the Model-View-Controller pattern with MongoDB integration.

üéØ What You'll Learn

  • MVC Architecture - Clean separation of concerns
  • MongoDB Integration - CRUD operations with a NoSQL database
  • Server-side Rendering - Traditional page-based navigation
  • REST API Design - Building RESTful endpoints
  • Form Handling - Creating, updating, and deleting records

üèóÔ∏è The MVC Pattern (jPulse Hybrid Approach)

Important: This demo uses jPulse's Hybrid MPA approach, where:

  • Model lives on the server (MongoDB operations)
  • View logic lives in the browser (DOM updates, interactivity)
  • Controller lives on the server (REST API endpoints)

This differs from Traditional MPA where all three (M, V, C) live on the server. See MPA vs SPA documentation for detailed comparison.

Model (site/webapp/model/helloTodo.js)

The Model handles all data operations on the server:

  • Database schema definition
  • CRUD operations (Create, Read, Update, Delete)
  • Data validation and business logic
  • MongoDB collection management

View (site/webapp/view/hello-todo/*.shtml)

The View handles presentation in the browser:

  • HTML templates with Handlebars syntax (server-rendered)
  • User interface layout and styling
  • Data display and formatting
  • Client-side interactivity (JavaScript in browser)

Controller (site/webapp/controller/helloTodo.js)

The Controller coordinates everything on the server:

  • Request routing and handling
  • Calling Model methods for data
  • Preparing data for Views
  • REST API endpoints (returns JSON)

üîÑ MPA vs SPA

This is a Multi-Page Application (MPA):

  • ‚úÖ Each page is a full HTML document with embedded JavaScript for interactivity
  • ‚úÖ Browser initiates REST API calls to the server
  • ‚úÖ Browser navigates between pages (page reloads)
  • ‚úÖ Server renders HTML for each page request
  • ‚úÖ Simple, traditional architecture
  • ‚úÖ Great for SEO and simple to complex workflows
