## Quick Start

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Start development:**
   ```bash
   npm run dev
   ```

3. **Visit your site:**
   - Main site: http://localhost:8080/
   - Admin: http://localhost:8080/admin/

## Production Deployment

For production deployment, this site includes a complete deployment package:

1. **Review deployment configuration:**
   ```bash
   cat deploy/README.md
   ```

2. **Setup system dependencies (as root):**
   ```bash
   sudo ./deploy/install-system.sh
   ```

3. **Configure environment:**
   ```bash
   # Review and customize .env file
   nano .env
   ```

4. **Setup database (as application user):**
   ```bash
   source .env && ./deploy/mongodb-setup.sh
   ```

5. **Start production:**
   ```bash
   npm run prod
   ```

## Site Customization

### Directory Structure

```
my-site/
├── webapp/                # Framework files (managed by jpulse-sync)
├── site/webapp/           # Your customizations
│   ├── app.conf           # Site configuration (generated by jpulse-setup)
│   ├── controller/        # Custom controllers
│   ├── model/             # Custom models
│   ├── view/              # Custom views
│   └── static/            # Custom assets
├── deploy/                # Production deployment package
│   ├── README.md          # Deployment guide
│   ├── install-system.sh  # System setup script
│   ├── mongodb-setup.sh   # Database setup script
│   ├── nginx.prod.conf    # nginx configuration
│   └── ecosystem.*.js     # PM2 configurations
├── .env                   # Environment variables (generated)
├── package.json           # Dependencies
└── README.md              # This file
```

### Creating Overrides

**Controllers:** Create `site/webapp/controller/[name].js`
**Views:** Create `site/webapp/view/[path]/[name].shtml`
**Assets:** Place in `site/webapp/static/[path]/`

## Updating Framework

```bash
# Current (private repo)
npm update @peterthoeny/jpulse-framework
npx jpulse-sync

# Once public: npm update @peterthoeny/jpulse-framework && npx jpulse-sync
```

> **Note**: Authentication setup is only required during initial installation while the repository is private.
